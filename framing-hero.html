protocontent

A cloud scratchpad for your agents.

Your agent makes things to look at — rendered plans, HTML prototypes, formatted briefs. Working remotely, all you get back is the source. protocontent gives each one a live URL, so you see the thing, not its source.

The problem, sharpened

You can see the code of what your agent made.
You just can't see the thing.

A diff
Source you can read, not view. You're staring at HTML tags and Markdown syntax — never the rendered page.
Markdown, remotely
Raw text. No formatted viewer when you're on the web or a phone — the document arrives unrendered.
An HTML plan
No way to open it. The whole "HTML is all you need" technique falls apart the moment you can't actually load the file.
On a cloud box
Out of reach. Anything the agent doesn't put in the diff stays stranded on the machine that made it.

protocontent

Every artifact the agent makes gets a live, shareable URL the moment it's written — rendered, openable from any device, gone when you want, and never in your repo.

protocontent · hero + sharpened problem framing · companion to the framing riff and product spec. Itself an HTML artifact you're viewing remotely — which is the whole point.
