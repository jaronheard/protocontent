# protocontent — positioning & homepage copy

A panel of six world-class positioning and copywriting experts was assembled to
write the website hero heading (H1) and core product positioning for
protocontent. Each produced a take, then each reviewed all six and voted for the
best. This document records the result and the final recommendation.

## The product, in one line

Your coding agent already writes files. protocontent makes each artifact it
produces — plans, prototypes, dashboards, mockups, diagrams, screenshots — land
at a live, shareable URL the instant it's created: ephemeral, sandboxed, never
in your repo, viewable from any device, on one live session page. It's the
missing verb after *generate the HTML*. For developers running
cloud/background/remote/multi-agent coding agents, where the
make-it / look-at-it / react loop breaks the moment work leaves your machine.

## The vote

No expert voted for their own take — each judged honestly. Winner-votes ended in
a dead heat:

| Take (author) | H1 | Wins | Runner-up |
|---|---|---|---|
| **Harry Dry** | "Your agent makes a file. You get a link." | Dunford, Fried | Ogilvy |
| **Joanna Wiebe** | "Your agent made it. Now just look at it." | Schwartz, Harry Dry | Fried |
| **April Dunford** | "Your agent writes a file. It gets a URL." | Ogilvy | Dunford, Harry Dry |
| **David Ogilvy** | "Stop reading your agent's code. Start watching its work." | Wiebe | — |
| Jason Fried | "Your agent makes it. Now you can see it." | — | Schwartz, Wiebe |
| Eugene Schwartz | "Your agent makes it. See it at a URL. Instantly." | — | — |

Weighted (win = 2, runner-up = 1): **Dry 5, Wiebe 5, Dunford 4** — a true tie.

## What the panel agreed on

- **The structure is settled.** Every expert independently landed on
  *"Your agent [makes/writes] it → you [get/see] [a link/URL]."* — a two-beat,
  primitive-first cadence that mirrors the product's own identity (*it's just
  "write a file," except it lands at a URL*).
- **The fault line is mechanism vs. emotion.** Strategists (Dunford, Fried) back
  "You get a link" — clearest, most honest, and the only finalist whose subhead
  tells you *how* (`npx`, "no repo, no deploy"). Desire-channelers (Schwartz,
  Wiebe) back "Now just look at it" — the word *just* sells the relief.
- **Best subhead material is cross-pollinated:** Fried's positioning line
  ("The missing step after 'make me an HTML page'") + Dry's concrete
  `npx` / no-repo-no-deploy proof.

## Final recommendation

Tiebreak: lead with the mechanism-first take, because this is a developer
infrastructure primitive and clarity is the conversion lever for a skeptical
senior developer. The emotional take becomes the A/B challenger.

The panel then ran a second iteration to pull exact language from the source
material (the "Unreasonable Effectiveness of HTML" blog and protocontent's own
foundation/spec HTML), change the install line to "add protocontent to your
coding agent," and enumerate the artifacts as working material that lives outside
source control. All six again kept the H1 and converged on the subhead below.

### Primary hero

> # Your agent makes a file. You get a link.
>
> The missing step after "make an HTML file." Add protocontent to your coding
> agent, and whatever it builds — a rendered plan, an HTML prototype, a formatted
> brief, a mockup, a diagram — is live at a link the instant it's created.
> Outside version control. Never staged, never committed. Your laptop, your
> phone, anywhere.

Every phrase traces to source: "make an HTML file" and the missing-step framing
come from the blog; "A rendered plan. An HTML prototype. A formatted brief. A
mockup. A diagram." and "never staged, never committed" come from protocontent's
own HTML.

### Subhead variant — names the gap out loud

For a version that quotes the blog's own admission of the leftover chore:

> The missing step after "make an HTML file": "as long as you upload the HTML
> file, you can share the link easily" — so you do it by hand, every time. Add
> protocontent to your coding agent, and whatever it builds — a rendered plan, an
> HTML prototype, a formatted brief, a mockup, a diagram — is live at a link,
> outside version control, the instant it's created.

### A/B challenger (emotional H1)

> # Your agent made it. Now just look at it.

(Pairs with the same subhead.)

### Positioning statement

For developers running cloud, background, and remote coding agents, the
make-it / look-at-it / react loop breaks the moment work leaves your laptop —
review collapses into raw code and dead screenshots. protocontent restores it by
extending the one thing every agent already does: writing a file. Each artifact
lands at a live, shareable URL the instant it's created — interactive, viewable
from any device, updating in real time on one session page. Ephemeral and
sandboxed by default, never in your repo. Not a deploy platform, not a host —
the missing **publish** verb.

## Appendix — all six original takes

**April Dunford** — H1: *Your agent writes a file. It gets a URL.* — frames a
missing "publish" verb, deliberately not a deploy platform; subhead lists
artifacts and plants "even when the agent isn't on your machine."

**Eugene Schwartz** — H1: *Your agent makes it. See it at a URL. Instantly.* —
diagnosed a problem-aware, highest-sophistication prospect; channels the "I want
to SEE it" desire and refuses the burnt-out word "deploy."

**David Ogilvy** — H1: *Every file your agent makes, live at a URL the second
it's made.* — best single line in the field as an alt: "Stop reading your
agent's code. Start watching its work."

**Harry Dry** — H1: *Your agent makes a file. You get a link.* — clearest,
most human; subhead ships with `npx`, "no repo, no deploy."

**Joanna Wiebe** — H1: *Your agent made it. Now just look at it.* — leads with
the transformation; the word *just* sells effortless relief.

**Jason Fried** — H1: *Your agent makes it. Now you can see it.* — plain and
honest; owns the sharpest positioning alt: "The missing step after 'make me an
HTML page.'"
